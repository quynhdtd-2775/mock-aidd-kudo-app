# Eval Templates

Skeletons kaizen instantiates into `.claude/skills/<dir>/extensions/evals/` on first benchmark. Both files are user-owned — teams edit and extend them freely.

## cases.md

```markdown
# Eval Cases: tkm:<skill-name>

Generated by /tkm:kaizen on <date>. Add your own cases below — one `## Case` block each.
Optional per-case `runs: N` (default 1) to reduce variance on flaky cases.

## Case 1: <short-title>
- input: |
    <realistic user request, verbatim as a user would type it>
- expected-behavior: |
    <which workflow phases/gates must fire, output shape, what must NOT happen>
- mode: <flag or default — which documented mode this exercises>

## Case 2: <short-title>
...
```

Generation rules:
- 3–5 cases minimum; every documented mode/flag exercised at least once
- Include one edge case (ambiguous input, missing prerequisite) — gate/error-recovery behavior is where skills differ most
- `expected-behavior` must be checkable from the response text alone (the judge sees only text)

## rubric.md

```markdown
# Judging Rubric: tkm:<skill-name>

Score each dimension 1–5 per response.

| Dimension | Weight | 5 looks like | 1 looks like |
| --- | --- | --- | --- |
| Instruction adherence | 3 | Every mandated phase/gate visibly honored | Gates skipped or reordered |
| Output correctness | 3 | Matches expected-behavior notes | Wrong or missing core output |
| Error handling | 2 | Edge cases handled per skill's recovery paths | Blind continuation on bad input |
| Token economy | 1 | Concise, no redundant prose | Padded, repetitive |

Winner = higher weighted sum. Within 5% → tie.
```

Customize weights per skill: a safety-critical skill raises Error handling; a report-producing skill raises Output correctness.
